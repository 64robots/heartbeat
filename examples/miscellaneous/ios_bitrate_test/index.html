This example shows the possible issues you'll experience on iOS if you use mp3 files with different bitrates.

The example loads a 128kbps mp3 of a short loop. You can add convolution reverb by using a 112kpbs mp3 of an impulse response (IR), by using the same IR at a bitrate of 128kbps, or by using the IR in wav format.

If you use the 112kbps IR, this sometimes results in a very distorted, crackeled sound, probably caused by the mix of sample rates.

It doesn't happen if you use the 128kbps IR which has the same bitrate as the file of the loop, or if you use the IR in wav format.

This bug is not documented and possible only temporal, but just to be on the safe side it is best to stick with 1 single bitrate when targeting iOS.
